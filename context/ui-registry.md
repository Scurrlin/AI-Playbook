# UI Registry

Purpose: reusable visual and interaction patterns.

If the project has no UI: not applicable.

## Registry Rules

- Add an entry when a component, screen pattern, or interaction pattern becomes reusable.
- Capture what future work must match: spacing, typography, color roles, states, behavior, and important exceptions.
